# 系统设置 - 告警设置（对齐说明）

> 返回 [系统设置索引](../07-系统设置.md)

---

本文件用于把 Admin Web 中“告警设置（Settings → Alerts）”的产品拆分，与 Admin API 现有接口做映射，避免告警相关字段散落在多个文档里难以维护。

## 现状映射（以现有接口为准）

### 1) 告警阈值（Thresholds）

当前阈值配置位于 Dashboard 模块接口中（阈值属于“配置”，但历史原因挂在 `/dashboard/alerts`）：

- `GET /api/v1/admin/dashboard/alerts`（见 `wallet_docs/admin-api/srds/02-Dashboard.md` 的 `F-ADMIN-013`）
- `POST /api/v1/admin/dashboard/alerts`（见 `wallet_docs/admin-api/srds/02-Dashboard.md` 的 `F-ADMIN-014`）

建议在实现层面仍可保持路径不变，但在 SRD 维护上，将“告警阈值”视为系统设置的一部分（与 Admin Web 信息架构对齐）。

### 2) 通知通道与收件人（Recipients / Channels）

通知开关与收件人配置当前在系统配置接口中体现：

- `GET /api/v1/admin/settings/config`（`notification.*`，见 `F-ADMIN-053`）
- `PUT /api/v1/admin/settings/config`（更新 `notification.*`，见 `F-ADMIN-054`）

字段示例（以 `F-ADMIN-053` 返回为准）：

- `notification.email_enabled`
- `notification.webhook_enabled`
- `notification.webhook_url`
- `notification.alert_recipients[]`

---

## 与“告警中心”的边界

- “告警设置”解决：阈值/规则/通知策略（配置治理）。
- “告警中心”解决：告警闭环处理（Ack/指派/静默/恢复/复盘，见 `wallet_docs/admin-api/srds/19-告警中心.md`）。
