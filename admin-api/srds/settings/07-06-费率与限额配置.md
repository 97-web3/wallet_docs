# 系统设置 - 费率与限额配置（对齐说明）

> 返回 [系统设置索引](../07-系统设置.md)

---

本文件用于把 Admin Web 文档中的“费率与限额配置”主题，与 Admin API 中分散在不同模块的配置接口进行统一收口说明。

## 现状映射（以现有接口为准）

### 1) 提现审批：限额与审批规则

- `GET /api/v1/admin/withdraw/config`（见 `wallet_docs/admin-api/srds/04-提现审批.md` 的 `F-ADMIN-029`）

该接口覆盖：

- `approval_rules.*`（自动审批、分级审批阈值）
- `limits.*`（单笔/日限额等）
- `risk_thresholds.*`（风险分级阈值）
- `notifications.*`（大额/高风险告警开关与阈值）

### 2) Swap：费率与 Swap 限额

- `GET /api/v1/admin/swap/config`（见 `wallet_docs/admin-api/srds/swap/01-全局配置.md` 的 `F-ADMIN-038`）
- `PUT /api/v1/admin/swap/config`（见 `wallet_docs/admin-api/srds/swap/01-全局配置.md` 的 `F-ADMIN-039`）

该接口覆盖：

- `global_settings.*`（含 `fee_rate`、`fee_min_usd`、`min/max/daily_limit`）
- `risk_settings.*`（高额阈值与 2FA 阈值）

### 3) 全局用户限额（通用）

系统配置中也存在用户侧限额（用于统一治理/兜底）：

- `GET /api/v1/admin/settings/config`（见 `wallet_docs/admin-api/srds/settings/07-03-系统配置.md` 的 `F-ADMIN-053`）
- `PUT /api/v1/admin/settings/config`（见 `wallet_docs/admin-api/srds/settings/07-03-系统配置.md` 的 `F-ADMIN-054`）

字段示例（以 `F-ADMIN-053` 返回为准）：

- `limits.user_daily_withdraw_usd`
- `limits.user_single_withdraw_usd`
- `limits.auto_approve_max_usd`

---

## SRD 维护约定（建议）

- 单一来源：各模块“配置类字段”的权威定义以各自 SRD 接口为准；本文件仅做收口与导航，避免重复拷贝导致字段漂移。
- 可选后续重构：若后端后续统一配置治理，可新增 `/api/v1/admin/settings/fees`、`/api/v1/admin/settings/limits` 等聚合接口，但需要确保与现有接口保持兼容期。
