# 输入密码登录(已填写) - 软件需求规格说明书

## 1. 页面标识

| 属性 | 值 |
|------|-----|
| 页面ID | CW-031 |
| 页面名称 | 输入密码登录(已填写) |
| 版本 | 1.0 |
| 更新日期 | 2025-12-08 |
| 状态 | 草稿 |

### 功能描述
输入密码登录已填写状态页面显示用户已输入手机号和密码的表单状态。登录按钮变为可点击的激活状态，等待用户提交登录请求。

---

## 2. UI组件

### 2.1 布局概述
页面延续登录页面布局，手机号和密码输入框都已填充内容，登录按钮从灰色禁用状态变为绿色可点击状态。

### 2.2 组件详情

| 组件 | 类型 | 描述 | 位置 |
|------|------|------|------|
| 返回按钮 | 图标按钮 | 左箭头 | 顶部左侧 |
| 页面标题 | 标签 | "登录" | 顶部中央 |
| 手机号输入框 | 文本输入 | 显示已输入的手机号，如"138****8888" | 表单区域 |
| 国际区号 | 标签 | 显示已选区号"+86" | 手机号左侧 |
| 密码输入框 | 密码输入 | 显示密码掩码圆点 | 手机号下方 |
| 显示密码按钮 | 图标按钮 | 眼睛图标 | 密码框右侧 |
| 忘记密码链接 | 文字链接 | "忘记密码？" | 密码框下方 |
| 登录按钮 | 主按钮 | 绿色激活状态，"登录" | 表单底部 |
| 注册链接 | 文字链接 | "还没有账号？立即注册" | 页面底部 |

## 3. 功能需求

### FR-031-01: 显示已填写状态

| 字段 | 内容 |
|------|------|
| **ID** | FR-031-01 |
| **描述** | 当用户填写完手机号和密码时，界面应显示已填写状态 |
| **验收标准** | 1. **假设** 用户输入完手机号，**当** 填写完成，**则** 输入框显示手机号<br>2. **假设** 用户输入完密码，**当** 填写完成，**则** 密码框显示掩码圆点 |
| **优先级** | 必须 |

### FR-031-02: 激活登录按钮

| 字段 | 内容 |
|------|------|
| **ID** | FR-031-02 |
| **描述** | 填写完必填字段后，登录按钮应变为可点击状态 |
| **验收标准** | 1. **假设** 手机号或密码为空，**当** 查看按钮，**则** 登录按钮为灰色禁用状态<br>2. **假设** 手机号和密码都已填写，**当** 验证通过，**则** 登录按钮变为绿色激活状态 |
| **优先级** | 必须 |

### FR-031-03: 允许修改输入

| 字段 | 内容 |
|------|------|
| **ID** | FR-031-03 |
| **描述** | 用户可以修改已输入的手机号或密码 |
| **验收标准** | 1. **假设** 已填写手机号，**当** 用户点击输入框，**则** 可以修改手机号<br>2. **假设** 已填写密码，**当** 用户点击密码框，**则** 可以修改密码 |
| **优先级** | 必须 |

### FR-031-04: 实时按钮状态更新

| 字段 | 内容 |
|------|------|
| **ID** | FR-031-04 |
| **描述** | 输入内容变化时应实时更新登录按钮状态 |
| **验收标准** | 1. **假设** 登录按钮已激活，**当** 用户清空密码，**则** 按钮立即变为禁用<br>2. **假设** 登录按钮禁用，**当** 用户补充完整信息，**则** 按钮立即变为激活 |
| **优先级** | 必须 |

## 4. 用户交互

### 4.1 用户工作流
1. 用户完成手机号和密码输入
2. 登录按钮变为激活状态
3. 用户检查输入内容
4. 用户点击登录按钮提交

### 4.2 导航流程
- **来源状态**: 表单填写中
- **目标状态**:
  - 点击登录: 提交验证
  - 修改输入: 保持在当前页面
  - 点击忘记密码: 跳转重置页面

### 4.3 手势与交互
| 手势 | 行为 |
|------|------|
| 点击输入框 | 修改已输入内容 |
| 点击登录按钮 | 提交登录请求 |

---

## 5. 数据需求

### 5.1 显示数据
| 数据字段 | 类型 | 来源 | 格式 |
|----------|------|------|------|
| 手机号 | String | 用户输入 | 部分掩码显示 |
| 密码 | String | 用户输入 | 圆点掩码 |
| 按钮状态 | Boolean | 表单验证 | 激活/禁用 |

### 5.2 输入数据
| 输入字段 | 类型 | 验证规则 | 必填 |
|----------|------|----------|------|
| 手机号(已填) | String | 格式正确 | 是 |
| 密码(已填) | String | 非空 | 是 |

---

## 6. 验证规则

### 6.1 按钮激活条件
- 手机号：非空且格式正确
- 密码：非空且长度符合要求

### 6.2 表单验证时机
- 实时验证输入内容
- 输入变化时立即更新按钮状态
