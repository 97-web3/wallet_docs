# 提交成功页面 - 软件需求规格说明书

## 1. 页面标识

| 属性 | 值 |
|------|-----|
| 页面ID | CW-020 |
| 页面名称 | 提交成功页面 |
| 版本 | 1.0 |
| 更新日期 | 2025-12-08 |
| 状态 | 草稿 |

### 功能描述
提币申请提交成功后的确认页面，显示成功状态和提示信息，提供查看交易记录和返回首页的操作。

## 2. UI组件

### 2.1 组件详情

| 组件 | 类型 | 描述 | 位置 |
|------|------|------|------|
| 成功图标 | 图标 | 黄绿色徽章图标，带勾选标记 | 页面上部中央 |
| 成功标题 | 文本标签 | "提交成功"黄绿色大号字体 | 图标下方 |
| 提示文字 | 文本标签 | "您的转账申请已提交，等待审核处理"灰色 | 成功标题下方 |
| 查看交易记录按钮 | 主按钮 | "查看交易记录"黄绿色按钮 | 中部 |
| 返回首页链接 | 文本链接 | "返回首页"黄绿色文字 | 底部 |

---

## 3. 功能需求

### FR-CW-020-01: 成功状态展示

| 字段 | 内容 |
|------|------|
| **ID** | FR-CW-020-01 |
| **描述** | 清晰展示提币申请提交成功的状态 |
| **验收标准** | 1. **假设** 提交成功, **当** 进入此页面, **则** 显示黄绿色成功图标和"提交成功"文字<br>2. **假设** 显示提示, **当** 渲染, **则** 告知用户"等待审核处理" |
| **优先级** | 必须 |

### FR-CW-020-02: 查看交易记录

| 字段 | 内容 |
|------|------|
| **ID** | FR-CW-020-02 |
| **描述** | 用户可以查看交易记录 |
| **验收标准** | 1. **假设** 用户点击"查看交易记录", **当** 触发, **则** 跳转到交易记录列表页面<br>2. **假设** 跳转后, **当** 加载, **则** 可看到刚提交的交易（状态为"处理中"） |
| **优先级** | 必须 |

### FR-CW-020-03: 返回首页

| 字段 | 内容 |
|------|------|
| **ID** | FR-CW-020-03 |
| **描述** | 用户可以返回钱包首页 |
| **验收标准** | 1. **假设** 用户点击"返回首页", **当** 触发, **则** 跳转到钱包主页<br>2. **假设** 返回主页, **当** 加载, **则** 清空整个提币流程的页面栈 |
| **优先级** | 必须 |

### FR-CW-020-04: 自动跳转

| 字段 | 内容 |
|------|------|
| **ID** | FR-CW-020-04 |
| **描述** | 在此页面停留一定时间后自动跳转 |
| **验收标准** | 1. **假设** 进入成功页面, **当** 停留3-5秒, **则** 可选择性自动跳转到交易记录页面<br>2. **假设** 用户已点击按钮, **当** 跳转, **则** 取消自动跳转定时器 |
| **优先级** | 可以 |

---

## 4. 用户交互

### 4.1 用户工作流
1. 用户在确认页面提交提币申请
2. 提交成功后进入此页面
3. 看到成功提示
4. 可选操作：
   - 点击"查看交易记录"查看提交的交易
   - 点击"返回首页"回到钱包主页
   - 等待自动跳转（如有）

### 4.2 导航流程
- 确认提币页面 → 提交成功页面
- 提交成功页面 → 交易记录列表页面
- 提交成功页面 → 钱包主页

---

## 5. UI设计规范

### 5.1 成功状态样式
- 成功图标：黄绿色徽章，大尺寸
- 成功标题：黄绿色，大号字体
- 提示文字：灰色，中号字体

### 5.2 按钮样式
- 主按钮：黄绿色背景，圆角
- 文本链接：黄绿色字体，无背景
