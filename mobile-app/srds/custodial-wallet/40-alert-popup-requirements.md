# 提示弹窗 - 软件需求规格说明书

## 1. 页面标识

| 属性 | 值 |
|------|-----|
| 页面ID | CW-040 |
| 页面名称 | 地址已复制提示 |
| 版本 | 1.0 |
| 更新日期 | 2025-12-08 |
| 状态 | 草稿 |

### 功能描述
成功复制地址后显示的Toast提示弹窗，以半透明深色背景显示"地址已复制"文字和成功图标，提供即时的操作反馈，自动消失无需用户操作。

---

## 2. UI组件

### 2.1 布局概述
悬浮在页面上方的Toast提示框，深色圆角背景，左侧显示绿色对勾图标，右侧显示"地址已复制"文字。

### 2.2 组件详情

| 组件 | 类型 | 描述 | 位置 |
|------|------|------|------|
| Toast容器 | 容器 | 深色半透明圆角矩形背景 | 屏幕顶部或中上部 |
| 成功图标 | 图标 | 绿色对勾/√图标 | Toast左侧 |
| 提示文字 | 文本 | "地址已复制"，白色文字 | Toast右侧 |

---

## 3. 功能需求

### FR-040-01: Toast提示显示

| 字段 | 内容 |
|------|------|
| **ID** | FR-040-01 |
| **描述** | 复制操作成功后显示Toast提示 |
| **验收标准** | 1. **假设** 用户复制地址, **当** 复制成功, **则** 立即显示Toast提示框<br>2. **假设** Toast显示, **当** 出现, **则** 使用淡入动画效果<br>3. **假设** Toast显示2-3秒, **当** 时间到, **则** 使用淡出动画自动消失 |
| **优先级** | 必须 |

### FR-040-02: 自动消失

| 字段 | 内容 |
|------|------|
| **ID** | FR-040-02 |
| **描述** | Toast提示自动消失无需用户操作 |
| **验收标准** | 1. **假设** Toast显示, **当** 2-3秒后, **则** 自动淡出消失<br>2. **假设** 显示期间, **当** 用户不操作, **则** Toast自然消失不影响用户<br>3. **假设** 多个Toast, **当** 连续触发, **则** 排队显示或替换当前Toast |
| **优先级** | 必须 |

### FR-040-03: 不阻塞操作

| 字段 | 内容 |
|------|------|
| **ID** | FR-040-03 |
| **描述** | Toast显示期间不阻塞用户其他操作 |
| **验收标准** | 1. **假设** Toast显示, **当** 用户继续操作, **则** Toast不阻塞用户交互<br>2. **假设** Toast显示, **当** 用户切换页面, **则** Toast跟随或消失<br>3. **假设** Toast显示, **当** 用户点击Toast区域, **则** 不影响下方元素的交互（Toast应该是非阻塞的） |
| **优先级** | 必须 |

### FR-040-04: Toast位置

| 字段 | 内容 |
|------|------|
| **ID** | FR-040-04 |
| **描述** | Toast显示在合适的位置 |
| **验收标准** | 1. **假设** Toast显示, **当** 出现, **则** 显示在屏幕顶部或中上部，不遮挡重要内容<br>2. **假设** 有键盘或其他浮层, **当** Toast显示, **则** 位置自动调整避免遮挡<br>3. **假设** Toast显示, **当** 出现, **则** 水平居中对齐 |
| **优先级** | 必须 |

## 4. 用户交互

### 4.1 用户工作流

```
用户执行复制操作 → 复制成功 → 显示Toast提示 → 2-3秒后自动消失
```

### 4.2 导航流程

- **触发**: 任何成功的复制操作（地址、交易哈希、助记词等）
- **显示**: 悬浮在当前页面上方
- **消失**: 自动淡出，无需用户操作

### 4.3 手势与交互

- **被动显示**: 用户无需操作，自动显示和消失
- **不可交互**: Toast本身不可点击或交互
- **不阻塞**: 不影响用户继续操作应用

|--------|------|------|----------|------|

## 6. 验证规则

### 6.1 显示规则

- Toast必须在操作成功后立即显示
- 显示时长2-3秒
- 使用淡入淡出动画

### 6.2 位置规则

- 水平居中
- 垂直位置避免遮挡重要内容
- 距离顶部或底部有合适的边距

### 6.3 多Toast处理

- 同时只显示一个Toast
- 新Toast触发时替换当前Toast
- 或使用队列依次显示

---

## 7. 技术考虑

### 7.1 Toast组件设计

- 全局单例Toast组件
- 支持不同类型（成功/失败/警告/信息）
- 可配置显示时长
- 支持自定义文字和图标

### 7.2 动画效果

- 淡入：0.3秒
- 停留：2-3秒
- 淡出：0.3秒
- 使用CSS动画或动画库

### 7.3 z-index管理

- Toast的z-index应该足够高
- 确保显示在所有内容之上
- 但不阻挡系统级UI（如状态栏）

### 7.4 可访问性

- 提供屏幕阅读器支持
- Toast内容朗读给视障用户
- 使用ARIA属性标注

### 7.5 多场景复用

- 地址已复制
- 交易哈希已复制
- 助记词已复制
- 保存成功
- 操作成功
- 等其他提示场景
