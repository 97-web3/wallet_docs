## 3. 功能需求

### FR-CW-013-01: 提币方式展示

| 字段 | 内容 |

| 字段 | 内容 |
|------|------|

## 4. 用户交互

### 4.1 用户工作流
1. 用户从选择币种页面选择了某个币种
2. 以底部弹出层形式显示提币方式选择页面
3. 用户查看两种方式的特点
4. 选择合适的提币方式：
   - **内部转账**：无手续费，需审核，适合平台内转账
   - **链上转账**：支持多链，需手续费，无需审核，适合外部地址转账
5. 进入对应的提币流程

### 4.2 导航流程
- 选择币种页面 → 选择提币方式页面（底部弹出）
- 选择提币方式 → 内部转账页面（选择内部转账）
- 选择提币方式 → 选择网络页面（选择链上转账）

### 4.3 手势与交互
- **点击卡片**：选择对应的提币方式
- **点击背景**：关闭弹出层，返回上一页
- **向下滑动**：关闭弹出层

## 8. 无障碍访问要求

- [ ] 所有文本颜色对比度符合WCAG 2.1 AA标准（最低4.5:1）
- [ ] 按钮和可交互元素最小触摸目标为44x44pt
- [ ] 支持屏幕阅读器，所有UI元素有适当的无障碍标签
- [ ] 卡片有明确的role="button"和aria-label描述
- [ ] 标签（0手续费、需要审核等）有适当的语义标记
- [ ] 弹出层获得焦点，支持Escape键关闭
- [ ] 对比表格有适当的表头标记（th元素）
- [ ] 支持键盘导航（适用于平板/桌面设备）
