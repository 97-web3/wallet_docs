# Web3钱包页 - 软件需求规格说明书

## 1. 页面标识
- **页面名称**: Web3钱包欢迎页/钱包初始化页
- **页面ID**: WEB3_WALLET_001
- **页面描述/用途**: 应用首次启动时的欢迎页面,提供创建新钱包或导入已有钱包的入口

## 2. UI组件
### 2.1 布局概述
页面采用垂直居中布局,暗黑主题背景。顶部显示品牌插画和欢迎标语,中部为主要操作卡片,底部显示安全提示语。右上角有跳过按钮。

## 3. 功能需求
### 3.1 主要功能
- 为新用户展示钱包功能介绍
- 提供创建新钱包的入口
- 提供导入已有钱包的入口
- 支持跳过引导页(可能用于多页引导流程)
- 传达Web3钱包的核心价值(自主托管)

### 3.2 业务逻辑
- 首次启动应用时显示此页面
- 如果用户已创建或导入钱包,跳过此页面直接进入主界面
- 跳过按钮可能关联多页引导流程,显示剩余页数
- **跳过限制**（重要）：
  - 跳过按钮仅允许跳过产品介绍类引导页
  - 用户**必须**选择"创建新钱包"或"添加已有钱包"才能继续
  - 跳过按钮**不会**绕过钱包创建/导入流程
  - 跳过按钮**不会**绕过助记词备份流程
- 创建钱包流程:生成新的助记词和密钥对
- 导入钱包流程:支持多种导入方式(助记词、私钥、iCloud备份、只读模式)
- **多钱包支持**：已有钱包的用户可通过 设置 → 添加钱包 访问此页面添加新钱包

## 4. 用户交互
### 4.1 用户操作
| 操作 | 触发条件 | 响应 |
|-----|---------|------|
| 点击跳过按钮 | 单击 | 跳过当前引导页,显示下一页或进入应用 |
| 点击创建钱包卡片 | 单击 | 进入创建新钱包流程(助记词生成页面) |
| 点击导入钱包卡片 | 单击 | 进入导入钱包选择页面 |
| 左右滑动 | 滑动手势 | 可能用于切换引导页(如有多页) |

### 4.2 导航流程
- 欢迎页 → 创建钱包流程(助记词生成、备份、验证)
- 欢迎页 → 导入钱包页面(选择导入方式)
- 欢迎页 → 跳过 → 主页面(如已有钱包)或下一引导页

### 4.3 手势操作
- 单击:选择操作选项
- 水平滑动:可能支持多页引导切换

## 5. 数据需求
### 5.1 数据展示
| 数据字段 | 数据类型 | 描述 |
|---------|---------|------|
| 引导页码 | Integer | 显示当前引导页位置(跳过3表示还有3页) |
| 欢迎文案 | String | 显示欢迎标题和描述 |
| 操作选项 | Array | 创建和导入钱包的选项信息 |

### 5.2 数据输入
无直接数据输入,用户通过选择操作进入下一流程

### 5.3 数据来源
- 本地配置:检查是否已有钱包数据
- 应用内文案:多语言文案资源
- 引导流程配置:确定显示哪些引导页

## 6. 验证规则
### 6.1 输入验证
无用户输入,不适用

### 6.2 错误处理
- 无需特殊错误处理
- 确保按钮点击响应正常
- 如果跳过后应进入主页,需验证是否已有钱包

## 7. 技术考量
### 7.1 状态管理
- 引导页显示状态(是否首次启动)
- 引导页进度(当前第几页)
- 用户选择状态(创建或导入)
- 钱包存在状态(是否已有钱包)

### 7.2 API集成
- 无需外部API
- 本地存储检查:查询是否已有钱包数据

### 7.3 安全考量
- 此页面不涉及敏感数据
- 确保导航到创建/导入流程时使用安全通道
- 在创建钱包前不应收集任何个人信息
- **"跳过"按钮安全限制**（必须实现）：
  - 跳过按钮仅跳过产品介绍/功能说明类引导页
  - 跳过按钮**绝不能**绕过钱包创建、导入或助记词备份流程
  - 实现时确保跳过逻辑不会意外允许进入未完成钱包设置的状态

## 8. 设计规范
### 8.1 视觉设计
- 使用暗黑主题(深色背景)
- 主题色:亮绿色(#C5FF3D或类似色值)
- 品牌插画:3D风格,现代感
- 卡片设计:圆角矩形,深灰色背景
- 图标设计:简洁扁平,与品牌色一致

### 8.2 动画效果
- 页面进入:淡入动画
- 卡片点击:轻微缩放反馈
- 页面切换:左右滑动过渡
- 插画:可能有微动效果
