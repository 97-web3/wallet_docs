# 底部导航 - 软件需求规格说明书

## 1. 页面标识
- **页面名称**: 底部导航栏
- **页面ID**: BOTTOM_NAV_001
- **页面描述/用途**: 应用主导航组件,提供三大核心功能模块的快速切换入口

## 2. UI组件
### 2.1 布局概述
固定在屏幕底部的水平导航栏,包含三个等宽的导航项。采用暗黑主题,选中项使用亮绿色高亮显示。导航栏背景为深黑色,带有轻微圆角。

### 2.2 组件详情
| 组件名称 | 组件类型 | 位置 | 描述 |
|---------|---------|------|------|
| 导航容器 | 容器组件 | 屏幕底部 | 深色背景,圆角矩形容器 |
| 资产导航项 | 导航按钮 | 左侧 | 卡片图标+文字"资产",未选中状态(灰色) |
| 资产图标 | 图标 | 资产项上部 | 卡片/信用卡图标,线性设计 |
| 资产文字 | 文本标签 | 资产项下部 | "资产"文字标签 |
| Web3钱包导航项 | 导航按钮 | 中间 | 钱包图标+文字"web3钱包",选中状态(亮绿色) |
| Web3钱包图标 | 图标 | Web3项上部 | 钱包图标,填充设计,亮绿色 |
| Web3钱包文字 | 文本标签 | Web3项下部 | "web3钱包"文字标签,亮绿色 |
| 我的导航项 | 导航按钮 | 右侧 | 用户图标+文字"我的",未选中状态(灰色) |
| 我的图标 | 图标 | 我的项上部 | 用户/个人中心图标,线性设计 |
| 我的文字 | 文本标签 | 我的项下部 | "我的"文字标签 |

## 3. 功能需求
### 3.1 主要功能
- 提供三大核心模块的快速切换
- 清晰显示当前所在模块(视觉高亮)
- 固定显示,始终可访问
- 支持点击切换不同功能页面

### 3.2 业务逻辑
- 默认选中"web3钱包"页面
- 点击导航项切换到对应页面
- 选中项使用亮绿色高亮,未选中项显示灰色
- 切换时保持各页面的状态(不重新加载)
- 在页面滚动时可能隐藏/显示(可选)

## 4. 用户交互
### 4.1 用户操作
| 操作 | 触发条件 | 响应 |
|-----|---------|------|
| 点击资产 | 单击资产导航项 | 切换到资产页面,图标和文字变为亮绿色 |
| 点击web3钱包 | 单击web3钱包导航项 | 切换到web3钱包页面,图标和文字变为亮绿色 |
| 点击我的 | 单击我的导航项 | 切换到个人中心页面,图标和文字变为亮绿色 |
| 重复点击当前项 | 双击当前选中项 | 可能触发回到页面顶部或刷新 |

### 4.2 导航流程
- 任意页面 → 资产页面(展示各类资产和投资组合)
- 任意页面 → Web3钱包页面(主钱包功能)
- 任意页面 → 我的页面(设置、账户管理等)

### 4.3 手势操作
- 单击:切换页面
- 左右滑动:可能支持页面间的滑动切换(与底部导航联动)

## 5. 数据需求
### 5.1 数据展示
| 数据字段 | 数据类型 | 描述 |
|---------|---------|------|
| 当前选中项 | String/Integer | 标识当前激活的导航项(assets/web3wallet/profile) |
| 导航项配置 | Array | 各导航项的图标、文字、路由信息 |
| 角标数据 | Integer | 可能显示未读消息数(如适用) |

### 5.2 数据输入
用户选择的导航项索引

### 5.3 数据来源
- 本地配置:导航项配置
- 应用状态:当前页面/路由状态
- 后端推送:角标数据(如有)

## 6. 验证规则
### 6.1 输入验证
- 确保同一时间只有一个导航项被选中
- 验证导航目标页面存在且可访问

### 6.2 错误处理
- 页面加载失败:显示错误提示,保持导航栏可用
- 网络错误:不影响导航切换,仅影响页面内容加载

## 7. 技术考量
### 7.1 状态管理
- 当前激活的导航项状态
- 各页面的内部状态(切换时保持)
- 历史导航栈(支持返回操作)
- 导航切换动画状态

### 7.2 API集成
- 无需直接API集成
- 各子页面可能需要独立的API调用

### 7.3 安全考量
- 导航切换不应泄露敏感信息
- 确保页面切换时安全上下文正确传递
- 某些页面可能需要身份验证(如我的页面)

## 8. 设计规范
### 8.1 视觉设计
- 背景色:深黑色(#1A1A1A或类似)
- 选中颜色:亮绿色(#C5FF3D或类似)
- 未选中颜色:灰色(#666666或类似)
- 圆角:12-16px
- 高度:约60-70px
- 图标大小:24x24px
- 字体大小:12-14px
- 间距:均匀分布,等宽设计

### 8.2 动画效果
- 切换动画:颜色渐变过渡(200-300ms)
- 图标变化:填充/线性切换
- 页面切换:淡入淡出或左右滑动
- 点击反馈:轻微缩放或亮度变化

### 8.3 响应式设计
- 适配不同屏幕宽度
- 平板可能显示更宽的导航项
- 支持横屏模式

## 9. 可访问性
### 9.1 无障碍支持
- 为每个导航项添加语义化标签
- 支持屏幕阅读器
- 提供足够的触摸目标大小(最小44x44pt)
- 高对比度模式支持

## 10. 性能要求
### 10.1 响应时间
- 导航切换响应:< 100ms
- 页面切换动画:< 300ms
- 导航栏渲染:< 50ms

### 10.2 内存管理
- 切换页面时保持前一页面状态
- 合理管理页面缓存,避免内存泄漏

- [ ] 三个导航项均匀分布
- [ ] 选中项正确高亮显示亮绿色
- [ ] 未选中项显示灰色
- [ ] 点击导航项能正确切换页面
- [ ] 页面切换动画流畅
- [ ] 导航项图标和文字对齐正确
- [ ] 暗黑主题样式正确
- [ ] 在所有主要页面都能看到导航栏
- [ ] 导航状态与页面状态同步
- [ ] 重复点击当前项有适当响应
- [ ] 触摸目标大小符合人机工程学
- [ ] 在不同屏幕尺寸下正常显示
- [ ] 动画性能良好,无卡顿
- [ ] 支持无障碍功能
