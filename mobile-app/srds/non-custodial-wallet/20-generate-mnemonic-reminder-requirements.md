# 生成助记词提醒页 - 软件需求规格说明书

## 1. 页面标识
- **页面名称**: 生成助记词提醒页
- **页面ID**: generate-mnemonic-reminder
- **页面描述/用途**: 在生成助记词之前向用户展示安全提醒和最佳实践建议，确保用户了解助记词的重要性和保管方法

## 2. UI组件
### 2.1 布局概述
页面采用垂直居中布局，顶部有返回按钮，中间为安全图标、标题和说明内容，底部为操作按钮。整体采用深色主题，使用荧光绿/黄渐变盾牌图标作为视觉焦点，营造安全感。

### 2.2 组件详情
| 组件名称 | 组件类型 | 位置 | 描述 |
|---------|---------|------|------|
| 返回按钮 | 按钮（图标） | 顶部左侧 | 返回上一页面 |
| 安全盾牌图标 | 图标图像 | 上部中央 | 渐变色盾牌图标，象征安全保护 |
| 主标题 | 文本标签（大号） | 图标下方 | "备份助记词，保护钱包安全" |
| 说明文字 | 文本段落 | 标题下方 | 解释助记词的作用和重要性 |
| 重要提醒卡片 | 信息卡片 | 说明文字下方 | 黄色警告图标 + 重要提示文字 |
| 建议备份方法标题 | 文本标签 | 提醒卡片下方 | "建议备份方法" |
| 备份建议列表 | 文本列表 | 方法标题下方 | 三条备份建议，每条以破折号开头 |
| 生成助记词按钮 | 主要操作按钮 | 底部上方 | 荧光绿色背景，执行生成操作。**初始状态禁用，页面停留5秒后自动启用** |
| 稍后再生成链接 | 文本链接按钮 | 底部 | 荧光绿色文字，跳过当前操作。**点击后显示确认对话框** |
| 倒计时提示 | 文本标签 | 生成按钮上方 | 显示"请先阅读以上内容 (5秒)"，倒计时结束后隐藏 |
| 跳过确认对话框 | 模态对话框 | 屏幕中央 | 警告用户跳过备份的风险，包含"我了解风险"和"返回备份"按钮 |

## 3. 功能需求
### 3.1 主要功能
- 向用户展示助记词的重要性说明
- 提供助记词备份的最佳实践建议
- 警告用户助记词泄露的风险
- 提供继续生成或稍后再生成的选项
- 确保用户在生成助记词前理解安全要求

### 3.2 业务逻辑
1. **页面展示流程**：
   - 用户在创建新钱包流程中进入此页面
   - 显示安全图标和警告信息
   - 用户阅读安全建议
   - 用户选择"生成助记词"或"稍后再生成"

2. **内容展示**：
   - 主标题：强调备份的重要性
   - 说明文字：解释助记词是恢复钱包的凭证，只保存在用户设备
   - 重要提醒：警告"任何人，只要获取到助记词，就可以控制您的资产"
   - 备份建议：
     * 使用笔和纸，将助记词按顺序正确抄写
     * 将助记词保存在安全地方
     * 不可将助记词进行网络存储与传输

3. **操作选项**：
   - 点击"生成助记词"：进入助记词显示页面（需等待5秒阅读时间后按钮才可用）
   - 点击"稍后再生成"：**必须**显示确认对话框，内容包括：
     * 标题："确定要跳过备份吗？"
     * 警告文字："如果您的设备丢失或损坏，没有备份助记词将**永久丢失所有资产**，且无法恢复。"
     * 主按钮："返回备份"（推荐操作，绿色）
     * 次按钮："我了解风险，稍后备份"（灰色，需等待3秒后可点击）

4. **跳过后的限制**（重要）：
   - 未备份钱包禁止发送交易，显示提示"请先完成助记词备份"
   - 首页持续显示红色警告横幅"您的钱包尚未备份，存在资产丢失风险"
   - 每次启动应用时弹出备份提醒
   - 未备份状态下，钱包功能受限仅可接收代币

## 4. 用户交互
### 4.1 用户操作
| 操作 | 触发条件 | 响应 |
|-----|---------|------|
| 点击返回按钮 | 用户点击左上角返回图标 | 返回到上一个页面（可能是创建钱包页面） |
| 等待阅读时间 | 页面加载后自动开始5秒倒计时 | 倒计时结束后启用"生成助记词"按钮，隐藏倒计时提示 |
| 点击生成助记词按钮 | 用户在倒计时结束后点击按钮 | 导航到备份助记词页面，显示生成的助记词 |
| 点击稍后再生成 | 用户点击"稍后再生成"链接 | **弹出确认对话框**，要求用户确认了解风险 |
| 确认跳过备份 | 用户在对话框中等待3秒后点击"我了解风险" | 进入主页，标记为未备份状态，功能受限 |
| 取消跳过备份 | 用户在对话框中点击"返回备份" | 关闭对话框，保持在当前页面 |
| 阅读页面内容 | 用户浏览页面 | 无交互响应，提供信息展示 |

### 4.2 导航流程
```
生成助记词提醒页 → 点击返回 → 上一页面（创建钱包）
生成助记词提醒页 → 点击生成助记词 → 备份助记词页
生成助记词提醒页 → 点击稍后再生成 → 上一页面或主页（标记为未备份）
```

### 4.3 手势操作
- 支持垂直滚动查看完整内容（如果内容超出屏幕）
- 支持左滑返回手势（iOS）
- 支持系统返回手势（Android）

## 7. 技术考量
### 7.1 状态管理
需要管理的状态：
- 页面加载状态
- 用户是否查看过此提醒（用于后续判断）
- 创建钱包流程的上下文状态
- 导航状态

### 7.2 API集成
- **助记词生成**：准备在下一页调用BIP39库生成助记词
- **用户偏好存储**：记录用户是否选择"稍后再生成"
- **分析统计**：记录用户在此页面的停留时间和选择

### 7.3 安全考量
1. **教育用户**：
   - 清晰说明助记词的重要性
   - 强调助记词泄露的风险
   - 提供具体的备份建议
   - 警告不要进行网络传输

2. **界面安全**：
   - 确保警告信息醒目显示
   - 使用视觉元素强化安全意识（盾牌图标、警告色）
   - 文案清晰易懂，避免用户误解

3. **流程控制**：
   - 允许用户稍后生成，但标记钱包为未备份状态
   - 在后续使用中提醒用户完成备份
   - 重要操作前再次提醒用户备份

4. **防止跳过**（必须实现）：
   - **强制阅读时间**：页面加载后"生成助记词"按钮禁用5秒，显示倒计时
   - **跳过确认对话框**：点击"稍后再生成"必须弹出确认对话框
   - **二次延迟确认**：对话框中"我了解风险"按钮需等待3秒后才可点击
   - **功能限制**：跳过备份后禁止发送交易，仅可接收代币
