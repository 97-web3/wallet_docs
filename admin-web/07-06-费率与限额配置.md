# Fees & Limits Configuration (费率与限额配置)

> Back to [Admin Web Docs](./README.md) · Up to [System Settings Index](./07-系统设置.md)

---

## Scope

- Withdrawal fee configuration (fee rules by chain/token)
- Limits configuration (deposit/withdraw/transfer thresholds)
- Governance controls (reasons, approvals, audit logging)

---

## Withdrawal Fees (手续费配置)

- **Entry**: `/admin/settings` → Fees (手续费配置)
- **Risk**: Direct impact on customer cost; changes must be audited and potentially approval-gated.

### Fee Rule Model (Recommended Baseline)

| Field | Type | Notes |
|---|---|---|
| `chain` | enum | `TRX`, `ETH`, `BSC`, ... |
| `asset` | string | Token symbol |
| `fee_type` | enum | `fixed`, `percent`, `dynamic_gas` |
| `value` | string/number | Fee value; interpretation depends on `fee_type` |
| `min_fee` | string/number | Optional |
| `max_fee` | string/number | Optional |
| `enabled` | boolean | - |

## Limits (限额配置)

- **Entry**: `/admin/settings` → Limits (限额配置)
- **Goal**: Risk control and operational protection for deposits, withdrawals, and internal transfers.

### Limit Dimensions (Recommended Baseline)

- **By operation**: deposit / withdraw / transfer
- **By time window**: per-transaction, per-day (per user, per tenant)
- **Approval thresholds**: “auto-approve max” vs “requires approval”
- **Scope**: global defaults + tenant overrides (recommended for multi-tenant)

## Governance & Audit Requirements

- All changes require a **reason** and produce a **diff** in audit logs.
- Sensitive changes should support **approval workflow** (recommended): proposer/approver split.
