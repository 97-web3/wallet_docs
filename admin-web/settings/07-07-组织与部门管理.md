# 组织与部门管理（Org & Department）

> 返回 [Admin Web 文档](../README.md) · 上至 [系统设置索引](../07-系统设置.md)

---

## 目标

为企业场景提供可治理的组织结构能力，用于：

- 薪资发放的部门汇总与审批路由（HR → Finance → CFO）
- 报表按部门/业务单元聚合
- 数据范围（Data Scope）：限制管理员只能查看/操作所属部门/业务单元的数据（可选）

---

## 页面概览（建议）

- **路由 (UI)**：`/admin/settings/org`
- **受众**：`super_admin`（读写）、`auditor`（只读）、`hr_manager`（只读或受限写）

---

## 数据模型（建议）

```typescript
type DepartmentStatus = 'active' | 'archived';

interface Department {
  department_id: string;
  tenant_id?: string;
  name: string;
  parent_department_id?: string;
  path?: string;                 // 物化路径（可选）
  status: DepartmentStatus;
  created_at: string;
  created_by: string;
  updated_at: string;
  updated_by: string;
}

interface BusinessUnit {
  business_unit_id: string;
  name: string;
  status: 'active' | 'archived';
}
```

---

## 页面功能（建议最小集）

### 1) 部门树/列表

- 创建/编辑/归档部门
- 支持层级（树形）与搜索
- 归档部门默认不可用于新员工/新审批路由

### 2) 业务单元（可选）

- 用于报表与审批的“业务分区”（在人员管理文档中已有提及：`business_units[]`）
- 若暂不做多租户，可仅做“业务单元标签”供聚合统计使用

---

## 与其他模块的联动（必须写清）

- 员工目录：`wallet_docs/admin-web/18-员工目录与映射.md` 使用 `department_id`
- 转账/薪资：部门汇总展示与审批流（见 `wallet_docs/admin-web/transfer/02-审批流程.md`）
- 报表：部门维度导出（见 `wallet_docs/admin-web/08-报表与导出.md`）

---

## API 映射（建议，若 SRD 暂未提供）

> 当前 Admin API SRD 中未明确组织/部门端点；落地时建议新增并纳入审计与 reason。

- `GET /api/v1/admin/settings/departments`
- `POST /api/v1/admin/settings/departments`（需要 `reason`）
- `PUT /api/v1/admin/settings/departments/{department_id}`（需要 `reason`）
- `POST /api/v1/admin/settings/departments/{department_id}/archive`（需要 `reason`）
