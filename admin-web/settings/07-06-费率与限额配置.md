## 范围

- 提现费用配置（按链/代币的费用规则）
- 限额配置（充值/提现/转账阈值）
- 治理控制（原因、审批、审计日志）

---

## 提现费用 (手续费配置)

- **入口**: `/admin/settings` → Fees (手续费配置)
- **风险**: 直接影响客户成本；变更必须经过审计，并可能需要审批权限。

### 费用规则模型 (推荐基线)

| 字段 | 类型 | 备注 |
|---|---|---|
| `chain` | 枚举 | `TRX`, `ETH`, `BSC`, ... |
| `asset` | 字符串 | 代币符号 |
| `fee_type` | 枚举 | `fixed` (固定), `percent` (百分比), `dynamic_gas` (动态Gas) |
| `value` | 字符串/数字 | 费用值；解释取决于 `fee_type` |
| `min_fee` | 字符串/数字 | 可选 |
| `max_fee` | 字符串/数字 | 可选 |
| `enabled` | 布尔值 | - |

### UI 表格 (旧版参考)

```
┌──────────────────────────────────────────────────────────────────┐
│ 提现费用配置                                            [保存]   │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │

│ ├───────┼───────┼────────────┼───────────────┼─────┼─────┼───────┤

│ │ ETH   │ ETH   │ 动态Gas    │ Gas * 1.5     │  -  │  -  │ 开启  │
│                                                                  │

```

## 限额 (限额配置)

- **入口**: `/admin/settings` → Limits (限额配置)
- **目标**: 针对充值、提现和内部转账的风险控制和运营保护。

### 限制维度 (推荐基线)

- **按操作**: 充值 / 提现 / 转账
- **按时间窗口**: 每笔交易, 每日 (按用户, 按租户)
- **审批阈值**: “自动审批上限” vs “需要审批”
- **范围**: 全局默认值 + 租户覆盖 (多租户推荐)

### UI 表单 (旧版参考)

```
┌──────────────────────────────────────────────────────────────────┐
│ 限额配置                                                [保存]   │
├──────────────────────────────────────────────────────────────────┤
│ 充值限额                                                           │
│  最低金额: [10] USDT (或等值)                                      │
│  每笔最大: [无限制]                                                │
│  每日最大: [无限制]                                                │
│                                                                   │
│ 提现限额                                                           │
│  最低金额: [10] USDT (或等值)                                      │
│  每笔最大: [50000] USDT                                            │
│  每日最大: [100000] USDT                                           │
│  自动审批阈值: [1000] USDT (关联地址)                              │
│                                                                   │
│ 转账限额                                                           │
│  最低金额: [1] USDT                                                │
│  每笔最大: [100000] USDT                                           │
│  每日最大: [无限制]                                                │
└──────────────────────────────────────────────────────────────────┘
```

## 治理与审计要求

- 所有变更都需要一个**原因**，并在审计日志中生成一个**差异**记录。
- 敏感变更应支持**审批工作流**（推荐）：提议者/审批者分离。
