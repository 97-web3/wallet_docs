## Page Overview

- **Entry Route**: `/admin/settings`
- **Purpose**: Centralized administration for identity governance, configuration governance, operational alerts, and auditability.
- **Primary Modules**:
  - Personnel Management (internal staff / admins)
  - Role & Permission Management
  - System Configuration (incl. Token/Chain infra mappings)
  - Alert Settings
  - Audit Logs
  - Fees & Limits Configuration

## Backend Interface Mapping (Reference)

Use `wallet_docs/admin-api/srds/07-系统设置.md` as the authoritative source for current API field names, validation rules, and error codes.

Common endpoints (see SRD for full details):

- Admin personnel:
  - `GET /api/v1/admin/settings/admins`
  - `POST /api/v1/admin/settings/admins`
  - `PUT /api/v1/admin/settings/admins/{admin_id}`
  - `POST /api/v1/admin/settings/admins/{admin_id}/disable`
  - `POST /api/v1/admin/settings/admins/{admin_id}/reset-password`
  - `POST /api/v1/admin/settings/change-password`
- Roles & permissions:
  - `GET /api/v1/admin/roles`
  - `POST /api/v1/admin/roles/assign`
  - `GET /api/v1/admin/users/{uid}/permissions`
- System configuration:
  - `GET /api/v1/admin/settings/config`
  - `PUT /api/v1/admin/settings/config`
- Audit logs:
  - `GET /api/v1/admin/settings/audit-logs`
