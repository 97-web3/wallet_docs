# 工作台与待办中心（Ops Workbench）

> 返回 [后台管理前端目录](./README.md)

---

## 页面概述

- **页面路径**：`/admin/workbench`
- **权限建议**：`finance/hr/admin/ops`（按模块分配），`auditor`（只读）
- **目标**：把“待审批/待处理/待复核”的事项统一收口，减少在多个页面来回切换。

---

## 信息架构（建议）

工作台默认提供 4 个分区（可折叠）：

1. **待审批**：出金审批、薪资批次审批、Vault 调整审批
2. **待执行/待重试**：已批准但未执行的批次、执行失败需要人工介入的记录
3. **对账待处理**：对账差异（pending/processing）
4. **系统与渠道异常**：来自告警中心的 critical 未 Ack / 未恢复事项（快捷入口）

---

## 列表字段（统一事项模型）

```typescript
type WorkItemType =
  | 'withdrawal_approval'
  | 'payroll_approval'
  | 'vault_adjustment_approval'
  | 'batch_execution'
  | 'reconcile_diff'
  | 'alert';

type WorkItemStatus = 'pending' | 'in_progress' | 'blocked' | 'done';

interface WorkItem {
  id: string;
  type: WorkItemType;
  title: string;
  priority: 'low' | 'medium' | 'high' | 'critical';
  status: WorkItemStatus;
  owner?: string;
  created_at: string;
  due_at?: string;        // SLA/超时
  link: string;           // 跳转到对应详情页
  summary?: string;       // 金额/链/部门等摘要
}
```

---

## UX 关键点（建议）

- 默认排序：`critical/high` + 最近到期（SLA）
- 一键筛选：我的待办 / 全部待办 / 已超时
- 批量处理仅对“同类型同策略”的事项开放（例如批量审批），避免误操作
- 每个事项必须可追溯到业务详情页，并可查看审计日志/trace_id（如适用）
