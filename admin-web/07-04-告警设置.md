# Alert Settings (告警设置)

> Back to [Admin Web Docs](./README.md) · Up to [System Settings Index](./07-系统设置.md)

---

## Page Overview

- **Entry**: `/admin/settings` → Alerts (告警设置)
- **Purpose**: Configure detection rules and notification routing for operational and risk events (balances, transactions, system health).
- **Audience**: Admin / Ops (depending on organization)

---

## Alert Rule Model (Recommended Baseline)

| Field | Type | Notes |
|---|---|---|
| `name` | string | Rule display name |
| `type` | enum | `balance`, `transaction`, `system` |
| `condition` | object | Threshold, operator, window |
| `level` | enum | `WARNING`, `CRITICAL` |
| `channels[]` | array | `email`, `in_app`, `sms`, `webhook` |
| `recipients` | object | Roles, users, distro lists |
| `mute_window` | time range | Optional (non-critical only) |
| `enabled` | boolean | Switch |
| `created_at/by` | datetime/string | Audit |

## Security & Compliance Notes

- Notification destinations (emails/webhooks) are sensitive config; changes must be audited.
- Webhook URLs should support allowlists, signing, and secret rotation (recommended).
