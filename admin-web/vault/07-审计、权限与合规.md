# Vault / 审计、权限与合规

## 1. 权限模型（建议）

当前页面描述为 Admin / Finance 可访问，但企业资金管理建议至少细分到以下能力：

| 能力 | 说明 | 推荐权限 |
|------|------|----------|
| 查看余额 | 查看各链 Vault 余额与同步状态 | Finance / Admin / Auditor |
| 配置阈值 | 修改 low/critical 与通知 | Admin（或 Finance+二次审批） |
| 手动同步 | 触发余额同步任务 | Admin / Ops |
| 发起调整 | 提交调整申请 | Finance |
| 审批调整 | 审批/拒绝申请（2FA） | Admin / SuperAdmin |
| 地址变更 | 启用/切换 Vault 地址 | SuperAdmin（建议） |
| 执行控制 | pending 加速/取消 | SuperAdmin（强审计） |

> 若系统当前只有 Admin/Finance 两类角色，建议在文档中标注“推荐演进方向”，避免误判合规成熟度。

---

## 2. 审计事件（建议最小集合）

至少需要记录以下事件，且可导出/可检索：

- Vault 余额阈值变更：谁、何时、从什么改到什么、影响范围（链/币种）
- 通知配置变更：收件人/URL 变更
- 告警处理动作：Ack/指派/备注/关闭（见 `vault/10-告警与通知.md`）
- 余额调整：申请、审批、拒绝、完成、失败（含证据材料）
- 手动同步：触发人、结果、耗时、失败原因
- 地址变更：启用/禁用/删除（高风险）
- 链上执行控制：加速/取消（高风险）
- 访问日志（合规增强）：谁查看了哪些 Vault（可选，但建议）

---

## 3. 合规与留存（建议写入文档）

- **数据留存**：调整记录、阈值变更、执行控制日志建议至少保留 1-3 年（视监管要求）
- **证据链**：附件（转账凭证、对账文件）需要不可篡改存储与访问控制
- **导出**：支持按时间范围导出审计事件与调整历史（CSV/PDF）
